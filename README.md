# Теннисное табло (Tennis Scoreboard)

Веб-приложение для отслеживания счета в теннисных матчах в режиме реального времени. Проект реализован на Python с использованием HTTP-сервера

## Основные возможности
- **Создание матча:** Регистрация нового матча с валидацией имен игроков.
* **Подсчет очков:** Автоматический расчет очков (0, 15, 30, 40, AD), геймов, сетов и тай-брейков согласно правилам тенниса.
* **Просмотр матчей:** Список всех завершенных матчей с поддержкой пагинации.
* **Фильтрация:** Поиск матчей по имени игрока.
* **Хранение данных:** Сохранение истории матчей в базе данных MySQL.

## Технологический стек
- **Backend:** Python 3.12+ (стандартная библиотека `http.server` и `socketserver`).
- **ORM:** SQLAlchemy (работа с MySQL).
- **Валидация:** Pydantic v2 (DTO и валидация входных данных).
- **Шаблонизатор:** Jinja2.
- **Миграции:** Alembic.
- **Контейнеризация:** Docker & Docker Compose.
- **Веб-сервер:** Nginx (раздача статики и проксирование).
- **Тестирование:** Pytest.

## Структура проекта
- `server.py` — точка входа, маршрутизация запросов и запуск сервера.
- `src/controllers/` — логика обработки HTTP-запросов.
- `src/services/` — бизнес-логика: расчет счета (`score_service`) и управление матчами (`match_service`).
- `src/dao/` — (Data Access Objects) взаимодействие с базой данных.
- `src/dto/` — (Data Transfer Objects) структуры данных для передачи между слоями и валидация.
- `src/models/` — описание таблиц базы данных через SQLAlchemy.
- `templates/` — HTML-шаблоны Jinja2.
- `static/` — статические файлы (CSS, JS, изображения).
- `tests/` — юнит-тесты для логики подсчета очков.

http://213.171.29.209:8080/

### Через Docker
1. Убедитесь, что у вас установлен Docker и Docker Compose.
2. Склонируйте репозиторий.
3. Запустите проект:
   ```bash
   docker-compose up -d --build
   ```
4. Приложение будет доступно по адресу: `http://localhost:8080`
